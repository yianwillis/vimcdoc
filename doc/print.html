1. 简介 print-intro

在 MS-Windows 上 Vim 可直接在系统已安装的打印机上打印文本。在其他系统上，会先 生成 PostScript 文件。对支持 PostScript 的打印机，可直接发送文件打印，对其他类 型的打印机，需要借助 ghostscript 等程序转换处理，完成打印操作。 备注: 如果 :hardcopy 打印不可用，备用方案是用 :TOhtml 导出为 HTML，再用浏 览器上打开打印。 :ha :hardcopy E237 E238 E324 :[range]ha[rdcopy][!] [arguments] 打印 [range] 范围内的所有行 (缺省是整个文件)。 在 MS-Windows 上，会显示打印对话框，用来选择打印机和 纸张尺寸等。[!] 版本跳过对话框。这里，打印机由 'printdevice' 指定。如果 'printdevice' 为空，就使用 系统缺省打印机。 对于非 MS-Windows 的系统，先写入 Postscript 临时文件， 然后通过 'printexpr' 设置的命令打印。'printexpr' 里可 通过 v:cmdarg 获取 [arguments] 参数。如果未设置 'printexpr'，忽略 [arguments]。'printoptions' 可用于设 置纸张尺寸，是否双面打印等。 备注: 需要打印到 pdf 时，可通过 "ps2pdf" 等外部工具将 PostScript 转换为 PDF。 :[range]ha[rdcopy][!] >{filename} 功能同上，但不直接打印，而是保存 Postscript 文件 {filename}。 文件名支持展开 "%" 等特殊字符 cmdline-special 。 注意同名文件会被静默覆盖。 {仅当编译时加入 +postscript 特性才有效} 在 MS-Windows 上应改用打印机驱动程序的 "输出到文件" 功 能。 打印过程中会实时显示完成的页号和百分比。要中断打印，按中断键 CTRL-C (在 MS-Windows 系统上用 CTRL-Break)。 'printfont' 和 'printoptions' 选项控制打印输出。'printheader' 指定页眉格式。 打印文件会严格限定在指定的页边距 (margin) 内，不受当前窗口 'wrap' 或 'linebreak' 设置影响。'printoptions' 里的 "wrap" 项可以用来关闭回绕。 打印文本使用当前高亮色彩，但有以下修正: 1) 普通背景强制渲染为白色 (白纸的颜色)。 2) 白色文本和缺省前景统一转为黑色，保证可见！ 3) 'background' 是 "dark" 时，颜色自动加深，避免在白纸上过浅看不清。

3 PostScript 打印 postscript-printing

E455 E456 E457 E624 只要磁盘空间充足，生成 PostScript 文件当无问题。前提是运行时文件正确安装 (如果 能打开帮助文件，一般说明安装正常)。 PostScript 打印有若干限制: - 'printfont' - 忽略字体名 (固定使用 Courier 家族字体 - 所有 PostScript 打印机 都应自带) 但可指定字体大小。 - 'printoptions' - 生成的 PostScript 输出会强制双面打印设置，但最终是否生效取 决于打印机硬件。不支持的打印机多数会直接忽略。但少数打印机会完全无法打印。 - 8 位支持 - 尽管很多 8 位打印字符编码都有支持，仍存在部分字符无法打印的问题。 这取决于打印机内置字体是否包含该字符。缺失字符缺省会用倒转问号 (¿) 代替，如 果该字形也不存在，则替换为空格。安装新版 Courier 家族字体，可完整输出当前编 码里的所有字符。 - 多字节支持 - Vim 目前会试图将多字节字符转换为 'printencoding' 指定的 8 位编 码 (为空时默认使用 latin1)。无法转换的个别字符会显示为未知字符。如果 Vim 完 全无法完成编码转换，整个打印过程会失败。

4. 自定义 8 位打印字符编码 postscript-print-encoding

E618 E619 如需打印 8 位字符数据时使用自定义打印字符编码，需要自行编写 PostScript 字体编 码向量。编码向量的完整编写细节超出本帮助文件的范围。可查阅 Addison-Wesley 出版 的 PostScript 语言参考手册第三版，或从 https://www.adobe.com/ 获取 PDF 版本。 以下仅说明使 Vim 识别并加载自定义打印字符编码的步骤。 i. 为编码向量定义唯一名称，以不与 Vim 已识别或标准编码名 (编码名参见 encoding-names ) 冲突、也不易和他人重复为原则。 ii. 将 $VIMRUNTIME/print/latin1.ps 复制到用户 'runtimepath' 里的 "print" 子目 录下，并重命名为自定义编码名。 iii. 编辑该文件。将所有 latin1 替换为自定义编码名 (包括 %%Title: 开头的那行)。 同时，将字形名 (glyph names) 数组替换为新编码向量。该数组必须恰好有 256 项，否则打印会直接失败！ iv. 在 Vim 里，将 'printencoding' 设为自定义编码名。 设置完成后，文件打印就会使用自定义打印字符编码。 除了 %%Title: 开头一行里的编码名和 %%Version:: 开头一行里的版本外，禁止修改前 三行其他任何内容，包括行顺序，否则 Vim 会报告该资源文件有错。 [PostScript 专业技术解释 - Vim 打印时，会根据 'printencoding' 值匹配同名文件。 该文件会定义名为 /VIM-name 的 PostScript Encoding 新资源，其中 name 为 Vim 实 际使用的打印字符编码。]

5. PostScript 中日韩打印 postscript-cjk-printing

E673 E674 E675 Vim 支持打印中日韩文本文件。相比普通英文打印，需要额外设置更多选项才能正常输 出。 中日韩各有多套标准字符集和编码，打印时必需同时指定两者。另外，这些字体通常没有 独立斜体字形，需要依赖字重 (weight) 和笔画粗细 (stroke style) 区分强调效果。因 此，因此，指定打印字体需要不同的设置逻辑。 'printencoding' 和 'printmbcharset' 选项分别用于指定编码和字符集。省略 'printencoding' 时，默认如常使用 'encoding'。手动指定 'printencoding' 后，打印 时字符会先被转换为该编码。必须确保该编码和文件内容所需字符集兼容，否则部分字符 可能无法打印。 'printmbfont' 指定打印所需中日韩字体。用于分别设置普通、斜体、粗体和粗斜体四种 语法高亮风格对应的打印字体。 Vim 不自带中日韩字体。部分免费中 (繁体) 日韩字体可从以下资源获取: https://resources.oreilly.com/examples/9781565922242/ 字体说明文档 https://resources.oreilly.com/examples/9781565922242/-/blob/master/00README 字体安装方法请阅读打印机文档。 中日韩字体可以非常巨大，因为包含数以千计的字形，很多字体只包含了国家标准的子 集。如果发现打印不出半角罗马字符，需要在 'printmbfont' 里设置对 ASCII 编码区使 用 Courier 字体。如果打印不出其他字符，请自动寻找包含所需字形的字体。 ASCII 字符的另一个问题是若干国家字符集对 ASCII 编码区指定不同字形。用国家字符 集打印 ASCII 文本时，可能不符预期。为此需要在 'printmbfont' 里设置对 ASCII 编 码区使用标准 ASCII 字符字形。 支持自建多字节字符集，但不建议新手操作。具体细节超出本帮助文件的范围，可见 https://www.adobe.com 提供的 PDF 文件格式官方文档 'Adobe CMap and CIDFont Files Specification, Version 1.0' 里的 CMap (字符映射表) 相关文件。

6. PostScript 打印疑难解答 postscript-print-trouble

E621 打印 PostScript 出现问题的通常唯一症状是完全没有打印输出。少数情况下会打印出一 张报错页，标明引发打印任务中断的 PostScript 操作符。 打印失败的若干可能原因: - prolog 资源文件版本错误。prolog 资源文件存放 Vim 打印必需的 PostScript 脚 本。不同 Vim 版本对应不同专属版本。请确保安装了正确的运行时文件，且在用户 'runtimepath' 里的 "print" 目录下，没有存放旧版本 prolog 文件。 - 纸张尺寸不兼容。部分 PostScript 打印机会因为不支持请求的纸张尺寸而中止打印。 Vim 缺省用 A4 纸。请确认实际打印机纸张规格，并相应设置 'printoptions' 的 "paper" 项。如果无法确定规格名，可实测打印纸尺寸，对照 'printoptions' 列出的 纸张尺寸列表，选取长宽 最 接近的规格。 注意: 实测纸张尺寸与文档标注会存在微小偏差。如果确实没有相近规格，可使用以下 讨论的 PSUtils 里的 psresize 工具缩放文件。 - 双面打印 (duplex)。多数不支持双面打印的 PostScript 打印机会自动忽略双面打印 请求。但部分设备会中止打印任务。可尝试关闭 "duplex" 项再打印。 注意: 可使用 PSUtils 工具手动实现双面打印 - 见下。 - 逐份打印 (collated)。和双面打印情形类似，多数不支持逐份打印的打印机会自动忽 略此种请求。但部分设备会中止打印任务。可尝试关闭 "collate" 项再打印。 - 语法高亮。部分打印管理代码不支持在黑白打印机上打印包含语法高亮的 PostScript 文件，即使全文只使用黑色。可尝试关闭 "syntax" 项再打印。 安全 'printoptions' 设置值为: :set printoptions=paper:A4,duplex:off,collate:n,syntax:n 将 "A4" 换成适配打印机的纸张规格。

8. 换页符 printing-formfeed

Vim 缺省对换页控制符 formfeed 不作任何特殊处理。设置 'printoptions' 的 "formfeed" 项可使 Vim 识别换页符，读到换页符立即换页，该行剩余内容在新页里继续 打印。换页符提供了较为原始的打印控制。但有若干注意事项。 只要一行包含换页符，Vim 会先输出这一整行 (开启行号时连带行号)。如果该行是页面 第一行且首字符就是换页符，会先打印空白页，再换到下一页输出内容。 因为行号只在包含换页符的行开头打印，新页打印的该行其余部分不再打印行号。这和 'printoptions' 里置位 "wrap" 项时，长行回绕行不显示行号的逻辑一致。 换页符在行尾时，新页内容会从第二行而非第一行开始。这是因为 Vim 处理完换页符后 再处理行尾，从而在继续打印之前，先自行下移一行。 根据以上几点，推荐使用换页符处理时，关闭行号打印，且不要将换页符放在行尾。还要 合理调整换页符上方行数，防止意外输出空白页。
