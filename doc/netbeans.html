1. 简介 netbeans-intro

NetBeans 接口最初是为了集成 Vim 到 NetBeans Java IDE，而使用外部编辑器插件开发 的。新版 NetBeans 民不再提供该配套插件，但该协议具有通用性，任意 IDE 均可基于 该协议集成 Vim。 Vim 的 NetBeans 协议是一个建立在传统的 TCP 套接字接口之上的基于文本的通信协 议。它不依赖于 Java 或 NetBeans 环境。任何语言或环境只要支持套接字接口，都可以 使用该协议控制 Vim。该协议已经有 C、C++、Python 和 Java 的实现。协议名称保留 NetBeans，只是基于历史渊源。 基于 Vim 的 NetBeans 协议的活跃项目: - Eclim, (死链) eclim.org/ VimIntegration，Vim 集成相关项目汇总: (死链) www.freehackers.org/VimIntegration 基于 Vim 的 NetBeans 协议的其他项目有: - Agide，Python 编写的用于 AAP 项目的 IDE (现已被 :Termdebug 替代): http://www.a-a-p.org (死链) - Clewn，C 编写，用于在 Vim 内集成 gdb: http://clewn.sourceforge.net/ - Pyclewn，Python 编写，用于在 Vim 内集成 gdb: http://pyclewn.sourceforge.net/ - VimWrapper，简化 IDE 接入 Vim 的工具库: (死链) freehackers.org/VimWrapper 已废弃项目 (链接不再有效): - VimPlugin，Eclipse 内集成 Vim 插件: http://vimplugin.sourceforge.net/wiki/pmwiki.php - PIDA，Python 编写，内置 Vim 的 IDE: http://pida.co.uk/ 相关项目集成 Vim 的用法请查阅项目相关页面。 可使用通道机制作为替代，见 channel 。 下文统一使用术语 "Vim 控制器"，代指通过 NetBeans 套接字接口控制 Vim 的程序。 关于 NetBeans IDE NetBeans 是一个开源的集成开发环境，由 Sun Microsystems, Inc 和 netbeans.org 开 发者社区共同开发。刚开始只是 Java IDE，后续发行版本陆续新增了 C、C++、和 Fortran 支持。 详见 NetBeans 主页 http://www.netbeans.org。 遗憾的是，外部编辑器模块现已宣布废弃。见 (链接看来已失效): externaleditor.netbeans.org。 Sun Microsystems, Inc. 也曾以 Sun ONE Studio 的名字发行 NetBeans。如需了解 Sun ONE Studio 系统产品线，可访问 http://www.sun.com。 当前 NetBeans 发行版本提供 Java 完全支持以及 C、C++ 以及 Fortran 的有限支持。 当前 Sun ONE Studio 发行版本提则完整支持 Java、C、C++ 和 Fortran。

2. 集成特性 netbeans-integration

Vim 的 NetBeans 套接字接口支持外部程序读取 Vim 信息或者指令 Vim 执行特定任务: - 获取缓冲区相关信息: 缓冲区名、光标位置、缓冲区内容等 - 接收缓冲区开关事件通知 - 接收缓冲区内容修改通知 - 读入或保存文件 - 修改缓冲区内容 - 安装专属键绑定 - 唤起 (raise) 窗口，控制窗口尺寸与位置 要发送键击到 Vim、或执行 Vim 函数功能，必须使用 clientserver 接口。

4. 错误消息 netbeans-messages

以下错误消息是 NetBeans 套接字协议特有的: E463 区域被保护，不能修改 Vim 控制器在文本中划定了受保护区域，该区域内容不允许编辑。必要 时同步锁定当前缓冲区。 E532 defineAnnoType 中的高亮颜色名太长 defineAnnoType 命令的 "fg" 或 "bg" 颜色参数仅 32 个字符。 2.5 版本新增。 E656 NetBeans 不允许写入未修改的缓冲区 禁止保存 Vim 控制器打开的未修改缓冲区。 E657 NetBeans 不允许缓冲区部分写入 Vim 控制器打开的缓冲区禁止局部写入。 E658 缓冲区 {缓冲区编号} 丢失 NetBeans 连接 Vim 控制器与当前文件状态同步错乱。为避免文件损坏风险，NetBeans 主动切断该文件的连接。此后文件的修改保存工作交由 Vim 接管，Vim 控制器将无法感知后续改动。 E744 NetBeans 不允许改变只读文件 Vim 通常允许编辑只读文件，仅在保存时报错。但 NetBeans 协议无法 同步处理对只读文件的编辑，这么做会造成状态错乱。因此，Vim 在运 行 NetBeans 协议模式时，禁止修改只读文件。

5. 从 NetBeans 模式启动 Vim netbeans-run

Vim 有两种方式进行 NetBeans 协议模式: + IDE 通过 -nb 命令行参数启动 Vim + 在 Vim 内部，可用 :nbstart 命令启动 NetBeans 连接 Vim 建立套接字连接时，使用 3 秒超时。 netbeans-parameters 设置 NetBeans 连接参数有三种方式。 从命令行启动时， -nb 命令行参数可以是: -nb={fname} 从文件读取配置 -nb:{hostname}:{addr}:{password} 直接给出完整连接信息 -nb 自动从配置文件或环境变量读取 从 Vim 内部启动时， :nbstart 的可选参数包括: ={fname} 从文件读取配置 :{hostname}:{addr}:{password} 直接给出完整连接信息 <无参数> 自动从配置文件或环境变量读取 E660 E668 从命令行启动时，出于安全考虑，推荐将配置信息写入仅用户可读的配置文件。 可通过 "-nb={fname}" 参数传递文件名。不带参数的 "-nb" 形式读取环境变量 "__NETBEANS_CONINFO" 指定的文件。配置文件必须包含以下三行内容，顺序不限: host={hostname} port={addr} auth={password} 忽略其余内容。Vim 控制器应负责在连接结束后删除该配置文件。 {hostname} 指定运行 Vim 控制器的机器名。省略时读取环境变量 "__NETBEANS_HOST" 值，环境变量不存在时默认使用 "localhost"。 {addr} 指定 NetBeans 协议端口号。省略时读取环境变量 "__NETBEANS_SOCKET" 值，环 境变量不存在时默认使用 3219。 {password} 指定连接密码。省略时读取环境变量 "__NETBEANS_VIM_PASSWORD" 值，环境 变量不存在时默认使用 "changeme"。 Vim 启动后会作为客户端，主动向指定终端和端口发起套接字连接。连接建立完成后，使 用 AUTH 事件发送密码，完成检验。

6. NetBeans 协议 netbeans-protocol

Vim 控制器和 Vim 使用明文消息通信。该协议最初为 NetBeans 外部编辑器设计。后来 经过扩展，适配 Agide (A-A-P 的 GUI IDE，以前位于 http://www.a-a-p.org，现在已 是死链)，以及其他各类 IDE。该扩展被标注为 "2.1 版本"。 协议 2.2 版本包含若干只影响 NetBeans 用户 (而非 Agide 用户) 的小改动。同时修复 一处容易引起混淆的缺陷: netbeans_saved() 函数旧版发送 "save" 协议指令。在 2.1 及更早版本里，被错误解读为文件已完成写入的通知。但这条指令实际是要求 NetBeans 保存文件，从而引发重复写入等各类异常问题。2.2 修正此问题，为避免再次混淆， netbeans_saved() 被改名为 netbeans_save_buffer()。 当前协议最新版本为 2.5。2.4 和 2.5 版本间的差异可在本文档中搜索 "2.5" 关键字查 看。 消息目前通过套接字传送。因为采用 UTF-8 明文，该协议理论上也可适配其他通信机 制。 Vim 仅在空闲等待用户输入时处理 Netbeans 消息。Vim 以非交互模式运行 (例如自动测 试脚本批量执行 Vim 脚本) 时，空闲循环触发次数可能不足，导致消息堆积。此时可在 脚本中调用 :sleep 命令。
