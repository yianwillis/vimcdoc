8. 目录栈 quickfix-directory-stack

快修机制会维护一套目录栈，用于保存从 make 输出中解析的各级目录。对 GNU-make， 目录管理非常简单，因为不管是通过 makefile 中的 'cd' 命令还是用 "-C dir" 参数 (读入 makefile 前进入指定目录)，每次进出目录都会输出绝对路径。推荐使用 '-w' 选 项，强制 GNU-make 在处理目录前后都输出当前目录。 非 GNU-make 则需要更复杂的目录维护逻辑。例如 AIX-make 从不输出当前目录的任何信 息。需要手动改造 makefile。Lesstif 的 makefile 提供命令，显示 "Making {target} in {dir}" 消息。但离开目录时仍不输出消息，而且也不输出绝对路径。 为解决相对路径和缺少 "退出目录" 消息的问题，Vim 使用以下算法: 1) 检查给定目录是否为当前栈顶目录的子目录。如果是则将其压入栈，成为当前目录。 2) 如果不是，则逐层向上遍历目录栈，检查是否为一级上级目录的子目录。 3) 仍无法匹配时，则假定为 Vim 当前工作目录的子目录。 此外还会校验每个文件是否在识别目录下真实存在。如果不存在，则逐层向上遍历目录栈 ( 并非 子目录树！)，检查文件是否位于上级目录。仍无法匹配时，则假定文件位于 Vim 当前工作目录。 此算法仍有局限。下例假定 make 只在进入目录时输出消息，消息类似于 "Making all in dir"。 1) 假定有以下目录和文件: ./dir1 ./dir1/file1.c ./file1.c 如果 make 在处理当前目录前先处理 "./dir1" 目录，而文件 "./file1.c" 报错， Vim 实际会加载文件 "./dir1/file.c"。 这只能靠输出 "退出目录" 消息解决。 2) 假定有以下目录和文件: ./dir1 ./dir1/dir2 ./dir2 会得到如下结果: Make 输出 Vim 推断的目录 ------------------------ ---------------------------- Making all in dir1 ./dir1 Making all in dir2 ./dir1/dir2 Making all in dir2 ./dir1/dir2 这里解决方法包括 "进入目录" 消息中输出绝对路径，或者输出 "退出目录" 消息。 通用的解决方案是确保输出绝对路径、以及输出 "退出目录" 消息。 Makefiles 改造示例 (在 Unix 上): libs: for dn in $(LIBDIRS); do \ (cd $$dn; echo "Entering dir '$$(pwd)'"; make); \ echo "Leaving dir"; \ done 然后将 %DEntering\ dir\ '%f',%XLeaving\ dir 加入 'errorformat' 选项，处理相应输出消息。 注意 Vim 并不校验 "退出目录" 消息中的目录名匹配栈顶当前目录。只需写 "Leaving dir" 消息即可，无需提供退出目录名。
