1. 介绍 popup-intro

本节所说的弹出窗口，是悬浮在普通窗口上层，由插件全权控制的浮动窗口。不能像普通 窗口一样，直接编辑弹窗文本。 弹窗可用于: - 临时显示信息，不挤占命令行区域 - 弹出交互对话框 - 键入时显示上下文提示信息 - 为自动补全提供额外信息 弹窗文本可用 text-properties 上色。也支持语法高亮。 缺省颜色取自三组高亮组，主体 hl-Popup 、边框 hl-PopupBorder 和标题 hl-PopupTitle 。为后向兼容，这些高亮组缺省都链接到 hl-Pmenu 组。它们可以被 重新定义，调整所有弹窗的外观，与补全弹出菜单有所区分。也可通过 "highlight" 参 数或 'wincolor' 选项单独设置单个弹窗的外观，例如: hi MyPopupColor ctermbg=lightblue guibg=lightblue call setwinvar(winid, '&wincolor', 'MyPopupColor') 弹窗中不显示 'hlsearch' 高亮。 弹窗和其他窗口一样，有自己的窗口 ID，但行为特殊。最大可以铺满整个 Vim 窗口，能 够覆盖其他窗口，弹窗间也能互相堆叠。堆叠层级由 "zindex" 属性控制。 E366 弹窗绑定专属缓冲区，也无法使用普通、可视或插入模式，不会接收键盘焦点。只能用 setbufline() 等函数修改专属缓冲区文本。弹窗及专属缓冲区还有更多有异于普通窗 口及缓冲区的行为，见 popup-buffer 。 如果弹窗功能不合所需，可查阅其他弹出机制: - 弹出菜单，见 popup-menu - 气泡，见 balloon-eval 窗 口 位 置 和 大 小 popup-position 弹窗高度默认等于缓冲区回绕后的行数。可由 "maxheight" 属性限制最大高度。可用插 入空行或 "minheight" 属性增加高度。 弹窗宽度默认等于缓冲区最长可见行长度。可由 "maxwidth" 属性限制最大宽度。可用补 空格或 "minwidth" 属性增加宽度。 缺省置位 'wrap' 选项，保证文本信息不会丢失。否则，如果空间不足，窗口会自动左移 以显示更多内容。右对齐窗口则会自动右移以显示更多内容。可设置 "fixed" 属性关闭 自动移位。 Vim 试图在指定位置显示弹窗。一旦弹窗可能会超出 Vim 主窗口边界，会自动就近显 示。例如， popup_atcursor() 通常在光标正上方显示弹窗，但如果光标靠近 Vim 窗口 顶部，会自动改为在光标下方放置。 Ex 命令输出导致屏幕向上滚动时，弹窗会跟随移动，避免遮挡命令输出。 光标即使被弹窗遮挡，仍然会显示。以便看清光标位置，但无法看到其所在文本。 关 闭 弹 出 窗 口 popup-close 通常逻辑是创建弹窗的插件负责自行关闭窗口。如果弹窗因故滞留，可关闭所有弹窗: call popup_clear(1) 通知等类型的弹窗会在指定时间后自动关闭。可在 popup_create() 中通过 "time" 属 性设置自动关闭时间。 未设置该属性的弹窗可设置 "close" 属性，控制手动关闭方式: 点击右上角 X、或点击 弹窗内部任意位置关闭弹窗。通知类弹窗 (见 popup_notification() ) 缺省已设置此 属性。 弹 出 缓 冲 区 和 窗 口 popup-buffer 调用弹窗创建函数时，如果直接传入文本内容，会创建新缓冲区保存文本与文本属性。此 缓冲区永久绑定本弹窗，且有诸多操作限制: - 无缓冲区名称 - 'buftype' 为 "popup" - 'swapfile' 关闭 - 'bufhidden' 为 "hide" - 'buflisted' 关闭 - 'undolevels' 为 -1: 完全禁用撤销 - 其他缓冲区局部和窗口局部选项均设为 Vim 缺省值。 虽然可以手动修改上列选项，但无法保证工作正常，建议保留原有设置。 弹窗本身具有光标位置，但不会显示。屏幕上显示的仍是底层窗口的光标，就像穿透弹窗 一样，方便用户看到实际文本的操作位置。 要在弹出窗口和缓冲区的上下文中执行命令，可用 win_execute() 。例如: call win_execute(winid, 'syntax enable') 要修改窗口选项，可用 setwinvar() ，例如: call setwinvar(winid, '&wrap', 0) 要修改缓冲区选项，可用 setbufvar() ，例如: call setbufvar(winbufnr(winid), '&filetype', 'java') 另一种方案是用 win_execute() 执行 :setlocal 命令。 弹 出 窗 口 中 运 行 终 端 popup-terminal 在弹窗中运行终端是个特例。许多规则会有变化: E863 - 终端窗口强制占据焦点，无法切换到其他窗口。 - 作业结束后，弹窗停留在终端-普通模式。可用 :q 关闭，或设置 "term_finish" 值 "close" 使作业结束后自动关闭 (见 term_start() )。 - 可用 popup_close() 关闭弹窗，终端缓冲区此时会被隐藏。 - 不可打开第二个终端弹窗。 E861 - 缺省 hl-Pmenu 色彩仅用于边框和内边距。要修改终端内部颜色，必须在创建终端前 先设置 hl-Terminal 高亮组。事后修改 'winhighlight' 也可生效，但需要终端内 程序全屏重绘。 - 缺省最小尺寸为 5 行 20 列；可通过 "minwidth" 和 "minheight" 参数自定义。 - 终端内程序输出文本时终端会自动扩容。可通过 "maxheight" 和 "maxwidth" 参数限 制最大尺寸。 要在弹窗中运行终端，可先创建隐藏终端，再将缓冲区号传给 popup_create()。示例: hi link Terminal Search let buf = term_start(['picker', 'Something'], #{hidden: 1, term_finish: 'close'}) let winid = popup_create(buf, #{minwidth: 50, minheight: 20})

"pos" 值设置。为 TRUE (缺省) 时， 如果弹窗无法在 "pos" 指定一侧完全显示，而在垂直方向另 一侧有更多可用空间，会在 "line" 行另一侧显示弹窗。 textprop 文本属性类型名，给出时弹窗会紧贴该类型的文本属性，并跟 随其移动。设为空串取消绑定。 详见 popup-textprop-pos 。 textpropwin 限定搜索文本属性的窗口。省略或无效值时使用当前窗口。仅 配合 "textprop" 生效。 textpropid 配合 "textprop"，用于根据属性 id 精准匹配唯一文本属 性。设为零解除绑定。 clipwindow 仅配合 "textprop" 生效。为 TRUE 时，弹窗被限制在文本属 性所属窗口内: 一旦文本属性滚动到窗口的上、下、左、右边
